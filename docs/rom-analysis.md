# ROM 分析记录 / ROM Analysis Notes

## 安全边界 / Safety boundary

中文：本仓库只提交分析工具、metadata 和文档。ROM 文件、提取图形、提取音乐、文本 dump、地图 dump 等 payload 必须保留在本地私有目录，不能提交或分发。  
English: This repository commits analysis tools, metadata, and documentation only. ROM files, extracted graphics, extracted music, text dumps, map dumps, and similar payloads must remain in local private directories and must not be committed or redistributed.

## 关键结论 / Key finding

中文：该 ROM 没有 CHR ROM，意味着图形不是一个独立、可直接切出的 CHR 区块。它使用 CHR RAM，真实 tile 通常在运行时从 PRG ROM 复制或解压到 PPU pattern table。  
English: The ROM has no CHR ROM, so graphics are not stored as a directly extractable CHR block. It uses CHR RAM; real tiles are typically copied or decompressed from PRG ROM into PPU pattern tables at runtime.

中文：因此，`extract-chr-candidates` 生成的 PRG-as-CHR sheets 只是人工观察线索，不应被视为最终素材提取结果。可靠方案是运行时追踪 PPU 写入。  
English: Therefore, PRG-as-CHR sheets generated by `extract-chr-candidates` are inspection hints only, not final extracted assets. The reliable approach is runtime PPU write tracing.

## 工具命令 / Tool commands

Inspect metadata and bank hashes / 检查 metadata 与 bank hash:

```sh
python3 tools/nes_rom_tool.py inspect '吞食天地2.nes' --out .omx/rom-analysis/metadata.json
```

Generate private candidate tile sheets / 生成私有候选 tile sheets:

```sh
python3 tools/nes_rom_tool.py extract-chr-candidates '吞食天地2.nes' --out-dir .omx/rom-analysis/chr-candidates
```

中文：输出默认写到 `.omx/`，该目录不入库。  
English: Outputs are written under `.omx/` by default, which is not tracked by git.

## 可提交 metadata / Commit-safe metadata

中文：`src/game/generated/rom-metadata.json` 只包含精简结构信息，用于 HUD 和工具状态显示，不包含 ROM payload 或提取素材。  
English: `src/game/generated/rom-metadata.json` contains reduced structural metadata for the HUD and tooling status display. It does not include ROM payload or extracted assets.

## 提取影响 / Extraction implications

1. 中文：必须支持 Mapper 74 的 bank switching。  
   English: Mapper 74 bank switching must be supported.
2. 中文：图形提取应基于运行时 PPU/CHR-RAM 写入追踪。  
   English: Graphics extraction should be based on runtime PPU/CHR-RAM write tracing.
3. 中文：文本提取需要通过字库 tile、引用地址和运行时显示路径发现 text table。  
   English: Text extraction requires discovering the text table through font tiles, references, and runtime rendering paths.
4. 中文：地图、事件、武将、战斗、物品是游戏特定格式，不能仅靠 iNES header 推导。  
   English: Maps, events, officers, battles, and items are game-specific formats and cannot be derived from the iNES header alone.

English: Build a runtime tracer that records CPU writes into PPU pattern tables, palettes, and nametables, then saves per-scene snapshots under `.omx/rom-analysis/runtime-captures/`.

## Runtime trace scaffold / 运行时追踪骨架

中文：`tools/nes_rom_tool.py trace-plan` 会生成 metadata-only 的运行时追踪计划，默认输出到 `.omx/rom-analysis/runtime-trace-plan.json`。该命令不提取 pattern table、nametable、palette、截图、文本或音频 payload；真正的 runtime capture 未来必须输出到被忽略的 `.omx/rom-analysis/runtime-captures/`。  
English: `tools/nes_rom_tool.py trace-plan` writes a metadata-only runtime tracing plan, by default under `.omx/rom-analysis/runtime-trace-plan.json`. It does not extract pattern tables, nametables, palettes, screenshots, text, or audio payloads; future runtime captures must be written to the ignored `.omx/rom-analysis/runtime-captures/` path.


## Town inference probe / 城镇推断探针

中文：`tools/nes_rom_tool.py town-probe` 会生成 metadata-only 城镇推断报告，列出候选 PRG bank 的 hash、entropy、结构比例等信息，但不输出原始字节、文本、图像、palette、nametable、pattern table、音频或 bank slice。当前小沛内部实现必须保持 `prototype` 标记，直到有更强证据。  
English: `tools/nes_rom_tool.py town-probe` writes a metadata-only town inference report with candidate PRG bank hashes, entropy, and structural ratios, but it does not output raw bytes, text, images, palettes, nametables, pattern tables, audio, or bank slices. The current Xiaopei interior must remain labeled `prototype` until stronger evidence exists.
