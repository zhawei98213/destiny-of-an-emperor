# Save Compatibility
# 存档兼容性

## Purpose
## 目的

Define how save compatibility is maintained while real content, chapter state, flags, inventory, and party state continue to evolve.
定义在真实内容、章节状态、flags、背包和队伍状态持续扩展时，如何维护存档兼容性。

## Current Compatibility Contract
## 当前兼容约定

- Old saves are accepted only through the migration runner.
- 旧存档只能通过迁移执行器进入当前运行时。
- The game runtime validates only the latest save schema after migration.
- 游戏运行时只校验迁移后的最新存档 schema。
- Save compatibility is owned by systems code, not UI code.
- 存档兼容性由系统层代码负责，不由 UI 层负责。

## What Must Stay Compatible
## 必须保持兼容的内容

### World State
### 世界状态

- map id
- 地图 id
- spawn point id
- 出生点 id
- player coordinates
- 玩家坐标
- facing
- 朝向
- step count when available
- 若存在则保留 step count

### Flags And Quest State
### Flag 与任务状态

- unknown old flags must not be silently invented
- 不能凭空补出未知旧 flag
- newly added flags default from content definitions
- 新增 flag 要从内容定义取默认值
- old quest state ids must still validate against current content when still present
- 旧任务状态 id 若仍存在，就必须继续通过当前内容校验

### Inventory
### 背包

- missing item arrays default to `[]`
- 缺失物品数组默认 `[]`
- missing gold defaults to `0`
- 缺失金钱默认 `0`
- migrated entries must still pass current item reference validation
- 迁移后的条目仍必须通过当前物品引用校验

### Party
### 队伍

- `partyMemberIds` must be preserved when present
- `partyMemberIds` 若存在必须保留
- missing `partyStates` default to `{}`
- 缺失 `partyStates` 默认 `{}`
- runtime can then rebuild missing member state from party definitions
- 之后由 runtime 根据队伍定义补齐缺失成员状态

### Chapter And Shop State
### 章节与商店状态

- chapter fields may be absent in old saves
- 旧存档里章节字段可以不存在
- shop state defaults to `{}`
- 商店状态默认 `{}`
- one-shot trigger consumption defaults to `[]`
- 一次性 trigger 消费状态默认 `[]`

### 样例 A：版本 1 -> 版本 3

## Regression Requirements
## 回归要求

Any save schema change must keep these checks green:
任何存档 schema 变更都必须保持以下检查通过：

- old version save can be loaded
- 旧版本存档可以读取
- migrated state is correct
- 迁移后状态正确
- missing fields receive reasonable defaults
- 缺失字段能得到合理默认值
- current version save still round-trips cleanly
- 当前版本存档仍能正常回写和回读

## Do Not
## 禁止事项

- do not branch on save version in scenes
- 不要在 scene 中按存档版本写分支
- do not patch save compatibility in menu UI
- 不要在菜单 UI 中修补存档兼容
- do not add a new save field without defining its legacy default
- 新增存档字段时不要不定义旧存档默认值
- do not change save structure without updating these docs
- 修改存档结构时不要不更新这些文档
